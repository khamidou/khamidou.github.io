A crash course in debugging production issues.


Last night something broke on a server. You got paged. After two hours of struggle you finally found that the issue was caused by a cron job which made a critical server run out of disk space. Sounds familiar? It certainly does for me.

You’re not going to get those hours of sleep back. However, you can learn how to solve this kind of problems faster. Debugging production issues isn’t something only UNIX greybeards can do --- it’s a skill you can learn, and not a hard one at that. If you’ve used a web framework like Rails before, you have the required troubleshooting skills.

In this free 5-lesson course, you'll learn about how to debug real-world, production issues. We’ll be going over the following topics:

  • Getting your ducks in a row: setting up your app for debugging
  • What to do when you get paged in the middle of the night
  • Network debugging: a whirlwind tour
  • Live debugging application code
  • Putting it all together
